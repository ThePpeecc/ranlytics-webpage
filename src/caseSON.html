Closing the loop on SON – by actually measuring and feeding back SON-driven changes in the RF domain continuously and in real time, and not by approximating their effects by measuring non-RF parameters

Implementing true service assurance models based on real-time measurements of actual RF performance rather than on inferences or estimates supports better management of SLA conformance

Closed-loop SON is a goal for many operators today.

Automatically receiving RAN parameters, analysing the data in comparison to normal traffic patterns & behaviours, and using the output to provision adjustments that are automatically & faultlessly applied to network and radio control equipment holds the prospect of improved network performance, better ROI, and lower operational costs.

Today, SON controllers collect data from BTSs, RNCs & other core IP network equipment, and apply complex, proprietary algorithms to infer what is happening in the RF domain. These inferences are used to drive the parameter changes that are then applied to BTSs and other control equipment.

Closed loop feedback is critical to a SON controller’s ability to determine the impact of changes that have been applied, and correcting and fine-tuning accordingly.

However, because actual RF data is not being measured and fed back to the SON controller, the feedback data that is being used today can only ever be a set of approximations. The resulting controller output can therefore only be approximate.

Also, the feedback data used today cannot reflect issues that can result from applied changes –such as cell handover problems, SiNR causing degraded user experience, etc.

For SONs to work effectively and realise operators’ goals, the effects of SON-driven changes to your RAN must be measured in the RF domain continuously, and fed back to the controllers in real time.

By deploying Stratus RF probes to continuously monitor RF parameters in real-time, and feeding that data back to the SON controllers, the feedback loop can be closed for the first time.
